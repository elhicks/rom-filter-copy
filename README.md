# rom_filter_copy

Filters a ROM collection by user rating and copies qualifying games to a target drive, preserving a full ES-DE compatible structure (ROMs, gamelists, and media).

Ratings come from `gamelist.xml` files generated by ES-DE's built-in scraper. Systems in the `copy_all_systems` list are copied in full regardless of rating — useful for older systems with small file sizes.

---

## Requirements

- Python 3.11 or higher
- ES-DE installed and a completed scrape of your ROM library
- ROMs organised in per-system subfolders (e.g. `ROMs/snes/`, `ROMs/psx/`)

---

## Finding your paths

You'll need three paths for the config file.

### ES-DE data directory (`esde_data_dir`)

This is where ES-DE stores its `gamelists/` and `downloaded_media/` folders.

| Platform | Default location |
|----------|-----------------|
| Windows | `C:\Users\<username>\ES-DE` |
| macOS | `/Users/<username>/ES-DE` |
| Linux | `/home/<username>/ES-DE` |

> **Windows users running this script under WSL:** translate drive letters to `/mnt/<letter>/`. For example, `C:\Users\Alice\ES-DE` becomes `/mnt/c/Users/Alice/ES-DE`.

### ROMs directory (`roms_dir`)

The folder that contains your per-system ROM subfolders. Example: if your SNES ROMs are at `F:\ROMs\snes\`, your `roms_dir` is `F:\ROMs` (or `/mnt/f/ROMs` in WSL).

### Target paths (`target_roms_dir`, `target_esde_data_dir`)

Where the script writes output. Two **independent** paths:

- `target_roms_dir` — where filtered ROMs land. The script creates `{target_roms_dir}/{system}/game.zip`.
- `target_esde_data_dir` — where ES-DE metadata lands. The script creates `{target_esde_data_dir}/gamelists/{system}/gamelist.xml` and `{target_esde_data_dir}/downloaded_media/{system}/{type}/...`.

Common case: both point at the same SD card root (e.g. `/mnt/g/ROMs` and `/mnt/g/ES-DE`). They can also live on different drives — set them to whatever paths your target device's ES-DE installation expects (check its `es_settings.xml`).

## Setup

1. **Scrape your ROMs.** Open ES-DE on your PC, go to **Scraper**, and run it against your ROM library. ScreenScraper is the recommended source. Wait for it to finish.

2. **Edit `config.toml`** in the same folder as the script:

```toml
roms_dir             = "/mnt/f/ROMs"                 # where your ROMs are
esde_data_dir        = "/mnt/c/Users/Alice/ES-DE"    # your ES-DE data directory

target_roms_dir      = "/mnt/g/ROMs"                 # where copied ROMs go
target_esde_data_dir = "/mnt/g/ES-DE"                # where copied gamelists + media go

rating               = 7.0                           # minimum rating out of 10

copy_all_systems = [                                 # copied in full, no rating filter
    "nes", "snes", "gb", "gbc", "gba",
    "megadrive", "gamegear", "mastersystem", "sega32x",
]
```

The two target paths are independent — they don't need to share a parent. Set
them to whatever your device's ES-DE configuration points at (see
`es_settings.xml` on the target).

3. **Run the script:**

```bash
python3 rom_filter_copy.py
```

The script previews everything that will be copied — number of games and total size per system — and asks for confirmation before touching any files.

By default, files that already exist on the target with the same name and size are **skipped**, so re-runs only write what's actually new. Gamelists are always rewritten (metadata may have changed even when media hasn't). Pass `--overwrite` to force a full re-copy.

---

## CLI flags

All config file values can be overridden at runtime:

| Flag | Description |
|------|-------------|
| `--target-roms-dir <path>` | Where copied ROMs go (e.g. `/mnt/g/ROMs`) |
| `--target-esde-data-dir <path>` | Where copied gamelists + media go (e.g. `/mnt/g/ES-DE`) |
| `--roms-dir <path>` | Override ROMs source directory |
| `--esde-data-dir <path>` | Override ES-DE source data directory |
| `--rating <0-10>` | Override minimum rating |
| `--systems <name> ...` | Only process specific systems, e.g. `--systems psx gc` |
| `--include-unrated` | Copy games that have no rating data |
| `--overwrite` | Force re-copy of files that already exist on target with matching size (default: skip them) |

---

## Output structure

```
{target_roms_dir}/
  psx/
    Crash Bandicoot (USA).zip

{target_esde_data_dir}/
  gamelists/
    psx/
      gamelist.xml
  downloaded_media/
    psx/
      covers/
      screenshots/
      ...
```

Point those config keys directly at the paths your device's ES-DE expects (typically `<sd-root>/ROMs` and `<sd-root>/ES-DE`, but check `es_settings.xml`). Artwork and metadata will load automatically.

## Behavior notes

**Unrated games are excluded by default.** Games with no `<rating>` element in `gamelist.xml` are treated as not meeting the threshold and are not copied. Pass `--include-unrated` to include them.

**`copy_all_systems` bypasses the rating filter.** Systems listed there are copied in full regardless of per-game ratings — intended for small or retro libraries where you want everything.

**Unsafe paths are silently dropped.** Any `<path>` element in a gamelist that is absolute (starts with `/`) or contains `..` is skipped. This is a guard against scraper bugs producing paths that could escape the ROMs directory.

**A malformed `gamelist.xml` skips that system, not the whole run.** If one system's gamelist fails to parse, a warning is printed to stderr and the script continues with the remaining systems. Other systems are unaffected.

**Re-running is safe.** Files already on the target with a matching name and size are skipped by default, so re-runs only write what's actually new. Gamelists are always rewritten so metadata changes propagate even when the media hasn't changed. Pass `--overwrite` to force a full re-copy regardless.

**Unknown `--systems` names warn but don't abort.** If you pass `--systems psx unknownsys`, a warning is printed for `unknownsys` and the run continues with the known names. If every name is unknown, the script errors out immediately.

**Ctrl-D at the confirm prompt aborts cleanly.** EOF (e.g. from a closed stdin or piped input that ends) is treated the same as answering "no" — nothing is copied.
